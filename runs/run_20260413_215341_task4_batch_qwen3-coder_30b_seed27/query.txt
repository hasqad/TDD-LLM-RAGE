Title: Library Book Sorting with Temporal Priority Rules

Description:
Given a list of library books with temporal metadata, implement a sorting algorithm that respects complex priority rules. Books must be sorted by: 1) Due date (earliest first), 2) Reservation priority (highest priority first), 3) Genre popularity (most popular first), 4) Book ID (lexicographically). However, books with overlapping reservation periods must be grouped together in the final sequence, and the algorithm must detect and report any circular reservation dependencies that would create an impossible scheduling scenario.

Function Signature:
def sort_library_books(books: list[dict]) -> list[str]:

Constraints:
Books list contains 1-1000 entries. Each book has 'id' (string), 'due_date' (integer timestamp), 'genre' (string), 'reservations' (list of [start_timestamp, end_timestamp] intervals), and 'priority' (integer). Reservation intervals may overlap. Due dates are unique. All timestamps are positive integers. Genre names are strings with length 1-50. Book IDs are unique strings with length 1-20.

Example Inputs:
[{'id': 'B001', 'due_date': 100, 'genre': 'Fiction', 'reservations': [[50, 80], [90, 110]], 'priority': 3}, {'id': 'B002', 'due_date': 120, 'genre': 'Science', 'reservations': [[60, 100]], 'priority': 1}, {'id': 'B003', 'due_date': 80, 'genre': 'Fiction', 'reservations': [[70, 90]], 'priority': 2}]

Expected Outputs:
['B003', 'B001', 'B002']
